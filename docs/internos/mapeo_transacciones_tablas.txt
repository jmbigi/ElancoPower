================================================================================
MAPEO DE REFERENCIA S/4HANA: TRANSACCIONES SAP A TABLAS
Resumen Ejecutivo: Mapeo de Transacciones y Tablas SAP para S/4HANA
================================================================================
Proyecto: Elanco Power
Fecha: 8 de noviembre de 2025
Propósito: Referencia técnica rápida para desarrolladores y analistas.
================================================================================

La confirmación de que el sistema fuente es **S/4HANA** define la estrategia de replicación. El foco principal es la tabla **`ACDOCA` (Universal Journal)**.

--------------------------------------------------------------------------------
SECCIÓN 1: EL FOCO PRINCIPAL - UNIVERSAL JOURNAL (FI/CO)
--------------------------------------------------------------------------------

| Módulo | Transacción | Tabla Clave S/4HANA | Observación |
| :--- | :--- | :--- | :--- |
| **FI/CO** | FAGLL03, KSB1, FB03, F.08, F.01 | **ACDOCA** (Universal Journal) | **CRÍTICA #1.** Contiene todas las partidas individuales y la información de la mayoría de las tablas clásicas (BSEG, FAGLFLEXA, COEP, etc.). **Replica solo esta en lugar de 10+ tablas antiguas.** |
| **FI/CO** | FAGLL03, F.08, F.01 | **ACDOCA_T** | Contiene los totales pre-agregados, muy útil para reportes de balance de comprobación. |
| **Maestros CO** | KSB1 | **CSKS, CSKA, AUFK** | Maestros de Centros de Costo, Elementos de Costo y Órdenes Internas. (Se mantienen iguales). |
| **FI Clásico** | FBL1N, FBL5N | **BKPF** | Cabecera de documento (aún necesaria). Las partidas (`BSID/K/A/S`) **NO son necesarias** si se replica `ACDOCA`. |

--------------------------------------------------------------------------------
SECCIÓN 2: MATERIALES Y LOGÍSTICA (MM/SD)
--------------------------------------------------------------------------------

| Módulo | Transacción | Tabla Clave | Propósito |
| :--- | :--- | :--- | :--- |
| **MM** | ME2L, ME23N | **EKKO, EKPO** | Cabecera y Posición de Pedidos de Compra. |
| **MM** | MB59, MB5B | **MKPF, MSEG** | Cabecera y Posición de Movimientos de Material. |
| **MM** | MM60 | **MBEW** | Valoración y Costo Estándar del Material (CRÍTICA). |
| **MM** | ZLEL008, MB5B | **MARD, MCHB** | Stock por Almacén y Stock por Lote (si se usa). |
| **SD** | VA05 | **VBAK, VBAP** | Cabecera y Posición de Órdenes de Venta. |
| **SD** | VA05 | **VBUK, VBUP** | **CRÍTICA #2.** Estatus de Cabecera y Posición de Documentos de Venta. |

--------------------------------------------------------------------------------
SECCIÓN 3: DATOS MAESTROS Y CONDICIONES
--------------------------------------------------------------------------------

| Módulo | Transacción | Tabla Clave | Propósito |
| :--- | :--- | :--- | :--- |
| **Maestros** | XK03, XD03 | **BUT000** | Tabla principal de *Business Partner* (BP). **CRÍTICA.** |
| **Maestros** | XK03, XD03 | KNA1, LFA1 | Tablas de clientes/proveedores (aún existen por compatibilidad). |

--------------------------------------------------------------------------------
SECCIÓN 4: TABLAS ESPECIALES (CLUSTER Y CUSTOM)
--------------------------------------------------------------------------------

| Módulo | Tabla | T-Code Asociado | Estrategia S/4HANA |
| :--- | :--- | :--- | :--- |
| **FI** | **BSEG** | N/A | **OBSOLETA** para replicación. La información está en **`ACDOCA`**. **No se debe replicar.** |
| **Global** | **STXL** | (Textos) | **Mantiene el estatus de Clúster.** Si necesita textos de documentos (ej. de pedidos), **SÍ debe replicar `STXL`** y aplicar la lógica de *declustering* en SLT. |
| **Custom** | ZLEL008, ZVEL015 | N/A | **ANÁLISIS ABAP OBLIGATORIO** para identificar las tablas `Zxxxx` subyacentes. |

--------------------------------------------------------------------------------
SECCIÓN 5: CONCLUSIÓN DEL MAPEO
--------------------------------------------------------------------------------

| Prioridad | Tablas / Estrategia | Razón |
| :--- | :--- | :--- |
| **CRÍTICA A** | Replicar **`ACDOCA`** y **`ACDOCA_T`**. | Sustituye a casi todas las tablas de Finanzas y Controlling (FI/CO) antiguas, resolviendo el problema de `BSEG` y `FAGLFLEXA`. |
| **CRÍTICA B** | Replicar **`VBUK/VBUP`** y **`MBEW`**. | Tablas esenciales para el estatus de documentos (SD) y la valoración de inventario (MM). |
| **ANÁLISIS Z** | Análisis ABAP de `ZLEL008` y `ZVEL015`. | Obtener la tabla **`Zxxxx`** subyacente para añadirla al alcance de SLT. |
| **CLUSTER** | Replicar **`STXL`** con *declustering*. | Si se necesita cualquier texto de documento de SAP. |
